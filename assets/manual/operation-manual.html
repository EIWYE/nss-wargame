第一章　平台概述与设计理念

1.1　平台定位与目标

国家安全战略兵棋推演平台是一套面向国家安全战略教育与训练的专业化兵棋推演系统。平台以"全域协同、智能裁决、实战导向"为核心原则，通过模拟真实国家安全场景，为战略分析人员、军事指挥员、外交官员和科研人员提供沉浸式的战略推演体验。

平台的核心目标是：

  • 战略思维训练：通过多域博弈场景训练用户的战略全局视野与跨域协同决策能力。
  • 态势感知提升：以全球多区域态势图和实时数据面板帮助用户建立对国际安全形势的立体认知。
  • 推演复盘分析：提供完整的推演记录与复盘报告系统，支持行动后反思与经验提炼。
  • 多角色协作：通过六大角色差异化系统，模拟真实组织架构中的分工协作场景。

1.2　核心设计理念

1.2.1　全域协同理念

现代国家安全博弈已从单一军事域对抗演进为多域同步博弈。平台采用六域模型——军事、经济、网络、外交、信息、国内，每个行动都会对多个域产生交叉影响。推演者必须在六个域之间平衡资源投入，理解跨域行动的时序配合与效果叠加原理。

1.2.2　智能裁决理念

平台采用百分骰系统作为裁决核心，结合多因素修正模型（力量战备修正、声望修正、国内支持修正、情报修正），在保证随机性的同时体现战略决策的质量差异。大成功（成功率×25%以下）将获得1.5倍效果加成，失败则仅获得30%效果并可能遭受反噬。

1.2.3　场景驱动理念

所有推演均围绕38个独立场景展开，每个场景拥有独特的背景设定、对弈双方、力量配置、情报包、专属行动和声望规则。场景之间通过8条推演链串联，模拟跨域连锁反应，使推演者体验从单一场景到全域联动的完整战略博弈过程。

3.2.1　全球态势图

  • 红色（5级）高烈度威胁——战争边缘态势或正在发生的冲突

用户可点击地图上的热点标记，查看对应场景的简要信息并直接进入推演。

4.4　跨区域联机对弈系统

跨区域联机对弈系统是平台的高级功能模块，通过WebSocket实时通信技术实现两位玩家跨浏览器进行真实的红蓝双方对弈。与单人推演模式不同，联机对弈中双方均由真实玩家操控，AI系统担任导调方负责态势推送、事件触发和骰子裁决，确保公平性。

4.4.1　系统架构

联机对弈系统采用客户端-服务器架构：

  • WebSocket服务器（server.js）：运行于9099端口，负责房间管理、玩家匹配、消息转发、AI导调逻辑和骰子裁决。所有游戏逻辑在服务器端执行，确保公平性。
  • 客户端模块（multiplayer.js）：嵌入平台前端，处理WebSocket连接、状态同步、UI交互和回调通知。支持自动重连（最多5次指数退避）和心跳测延迟。
  • UI层（app.js）：四阶段状态机驱动UI渲染——大厅阶段、房间阶段、游戏阶段、结果阶段。通过回调函数响应服务器消息实时刷新界面。

4.4.2　对弈流程

阶段操作说明
1. 大厅创建/加入房间玩家选择推演场景后创建房间（获得6位房间号），或输入房间号加入。支持公开房间快速匹配。
2. 房间阵营选择与准备双方玩家在房间大厅选择红方或蓝方阵营（根据场景SCENARIO_SIDES配置），选择完成后点击"准备就绪"。双方均准备后游戏自动开始。房间内支持实时聊天。
3. 游戏多轮对弈每轮流程：AI导调方推送态势事件→双方各自选择战略行动（消耗行动点）→双方提交行动→服务器掷骰裁决→双方查看裁决结果→进入下一轮。行动选项根据场景和阵营动态过滤（红蓝双方拥有不同的战略行动集）。
4. 结果终局结算达到最大轮次后游戏结束，显示最终比分、胜负判定和完整推演日志。支持投降和断线判负。

4.4.3　AI导调方功能

在联机对弈中，AI导调方承担以下职责：

  • 态势推送：每轮开始时推送场景相关的导调事件（台海、南海、经济、网络、边境等分类事件库），为双方提供决策背景。
  • 骰子裁决：双方提交行动后，服务器为每个行动掷百分骰，根据基础成功率（受升级度影响）判定大成功/成功/失败，计算本轮得分和六域效果变化。
  • 状态同步：实时同步六域状态、双方分数、升级度等游戏状态至双方客户端。
  • 终局判定：最大轮次结束后根据总分判定胜负，生成推演日志。

联机对弈服务器需独立启动：

  • 依赖安装：在nss-dashboard目录下执行 npm install ws
  • 启动命令：执行 node server.js，服务器监听9099端口（可通过WS_PORT环境变量配置）

第五章　推演引擎核心规则

推演引擎是平台的裁决核心，负责行动成功率计算、骰子判定、效果结算、资源消耗和综合评分。本章详细说明引擎的全部规则和公式。

5.1　百分骰系统

平台采用百分骰作为基础裁决机制。每个行动执行时，系统生成1-100的随机整数（骰值），与该行动的成功率进行比较：

骰子判定规则
结果条件效果倍率说明
大成功骰值 ≤ 成功率 × 0.25×1.5行动效果增强50%，额外正面影响
成功骰值 ≤ 成功率×1.0行动按正常效果执行
失败骰值 > 成功率×0.3仅30%效果，可能遭受反噬
大成功概率说明

大成功阈值为成功率的25%。例如：某行动成功率为80%，则大成功阈值为20（80×0.25=20），即骰值≤20时触发大成功，概率为20%。这意味着成功率越高的行动，大成功概率也越高——鼓励用户通过力量配置和情报准备提高行动成功率。

5.2　成功率计算公式

每个行动的最终成功率由基础成功率经过多因素修正后得出：

最终成功率 = 基础成功率 + 力量修正 + 声望修正 + 国内支持修正 + 情报修正 + 功能区加成 (上限100，下限5)

5.2.1　力量修正

当行动属于军事域时，系统计算当前场景所有军事域力量的平均战备度，修正公式为：

力量修正 = (军事力量平均战备度 - 50) × 0.3

例如：场景中有4个军事力量，平均战备度为85，则力量修正 = (85-50) × 0.3 = +10.5%。

5.2.2　声望修正

声望修正 = (当前国际声望 - 50) × 0.2

声望高于50时获得正向修正，低于50时受到负向修正。例如：声望为65时，修正 = (65-50) × 0.2 = +3%。

5.2.3　国内支持修正

国内支持修正 = (当前国内支持度 - 50) × 0.1

国内支持度影响较小（系数0.1），反映民意对行动执行力的间接影响。

5.2.4　情报修正

已揭示的情报卡中，与行动域匹配的情报项提供额外成功率修正。情报修正值由情报项的修正值字段决定（通常为+5%到+15%）。空间域情报自动映射为网络域修正。

5.2.5　功能区加成

推演前在功能区执行的准备行动可为推演提供额外加成，包括：成功率加成（+3%到+5%）、额外行动点、先手优势、防御加成等。具体加成取决于功能区行动的类型和数量。

5.3　资金系统

资金是推演中的核心战略资源，用于执行行动（资金消耗）和衡量经济状态。每个场景设有起始预算，推演过程中通过行动消耗和科技研发收益动态变化。

5.3.1　资金消耗

每个战略行动定义了资金消耗字段（单位：亿元），从0到2000不等。例如：

  • 外交声明：10亿元
  • 网络情报搜集：100亿元
  • 前沿军事威慑部署：800亿元
  • 科技自立攻关：2000亿元
  • 精确军事打击：1500亿元

5.3.2　科技研发收益

部分科技类行动执行后会在下一轮产生资金收益。收益金额由推演规则的科技研发收益映射表定义：

行动下轮收益（亿元）行动下轮收益（亿元）
科技自立攻关300人民币国际化推进200
芯片自主攻关350碳减排技术突破300
量子技术突破300能源转型推进300
供应链国产替代300稀土下游深加工250
跨境支付系统建设200基因数据自主平台250

5.3.3　经济基础收益

每轮结束时，如果经济域状态≥50，系统自动按当前资金的3%生成基础收益：

经济基础收益 = 当前资金 × 0.03 （仅当经济域 ≥ 50 时生效）

5.3.4　资金评分

推演结束时的资金评分采用20分下限公式：

资金评分 = 20 + 取大值(0, 剩余资金/起始资金) × 80 （结果限制在0-100之间）
设计说明

20分下限确保即使用户消耗了大部分预算（合理的战略行为），资金评分也不会归零。这避免了"不花钱=高分"的逆向激励，鼓励用户积极使用资源达成战略目标。

5.4　声望系统

国际声望反映中国在国际社会中的形象和影响力。不同场景类型适用不同的声望规则：

5.4.1　声望类型

类型适用场景战争行动惩罚倍率和平行动增长倍率
维护主权维护主权类（台海、南海、边境等）×1.5×1.0
侵略行动侵略行动类×0.5×1.0
科技竞争科技竞争类（量子、太空碎片、基因数据）×1.0×1.0
不适用犯罪/恐怖组织类（网络攻击、混合战争、认知战、生物安全）×0（不生效）×0（不生效）

5.4.2　战争行动与和平行动判定

和平行动判定：行动升级度 ≤ 0 且 行动声望效果 > 0 战争行动判定：行动升级度 ≥ 2

5.4.3　声望变化计算

战争行动声望变化 = 行动声望效果 × 战争惩罚倍率[场景类型] 和平行动声望变化 = 行动声望效果 × 和平增长倍率[场景类型]（下轮结算）
示例

在台海场景（维护主权类型）中执行"精确军事打击"（声望效果=-15，升级度=3）：

声望变化 = -15 × 1.5 = -22.5（即时扣除）

在印太场景（维护主权类型）中执行"高级别峰会"（声望效果=10，升级度=-2）：

声望变化 = 10 × 1.0 = +10（下轮结算增长）

5.5　升级阶梯与智能对手行为

升级度反映当前对抗的烈度水平，从1到5共5个等级。用户的军事行动和部分高烈度行动会增加升级度，外交谈判和和平行动可降低升级度。

5.5.1　升级度变化

每轮结束后，系统汇总所有行动的升级度值，计算该轮净升级度变化：

本轮升级度变化 = Σ(所有行动的升级度值) 当前升级度 = 取大值(1, 取小值(5, 上一轮升级度 + 本轮变化))

5.5.2　升级度对智能对手行为的影响

智能对手侵略性参数直接决定蓝方发起反制行动的概率：

智能对手反制概率 = 侵略性参数 × (0.5 + 随机扰动0.3)

在升级度3（中烈度）时，智能对手侵略性为0.65，反制概率约为32.5%-65%。在升级度5（高烈度）时，智能对手侵略性为0.95，几乎必定发起反制。

5.6　评分系统

推演结束后，系统根据五项加权指标计算综合评分（0-100）：

指标权重计算方式说明
六域状态35%六域状态平均值军事/经济/网络/外交/信息/国内六域的综合状态
国际声望20%最终声望值反映国际社会对中国行动的认可度
国内支持20%最终国内支持度反映国内民众对政策的支持程度
资金状态10%20 + 取大值(0, 剩余/起始)×8020分下限的资金评分
升级控制15%(5 - 最终升级度) × 20低升级度=高分，升级度5=0分
综合评分 = 六域均值 × 0.35 + 声望 × 0.20 + 国内支持 × 0.20 + 资金评分 × 0.10 + 升级控制 × 0.15
评分等级
  • 85-100：战略胜利——出色完成战略目标，全域优势明显
  • 70-84：战术胜利——达成主要目标，部分域有不足
  • 55-69：僵持——双方均未取得决定性优势
  • 40-54：战术失利——主要目标未达成，多域处于劣势
  • 0-39：战略失败——核心利益受损，局势失控

第六章　力量战备与情报系统

6.1　场景化力量配置

每个场景拥有独特的军事力量配置，替代了早期版本中所有场景共用同一套力量列表的设计。力量配置直接影响军事域行动的成功率修正。

6.1.1　力量数据结构

每个力量项包含以下字段：

字段说明
军种代码军种代码（陆军/海军/空军/火箭军/军事航天/网络空间/信息支援/联勤保障/武警/海警局/民兵预备役）
战备度战备度（0-100），影响军事域行动修正
状态状态（已部署/高度戒备/待命/活跃）
部署位置部署位置
装备装备描述
备注备注信息

6.1.2　力量对推演的影响

军事域行动执行时，系统自动计算场景中所有军事域力量的平均战备度：

军事力量平均战备度 = Σ(军事力量战备度) / 军事力量数量 力量修正 = (平均战备度 - 50) × 0.3

不同场景的军事力量配置差异很大。例如：

  • 台海场景：海军(88)、空军(92)、陆军(80)、火箭军(95)、军事航天(88)、网络空间(90)、信息支援(85)、联勤保障(82)、海警局(90)——全域力量齐备，平均战备度高
